11//! Generating random samples from probability distributions.
12//!
13//! This module is the home of the [`Distribution`] trait and several of its
14//! implementations. It is the workhorse behind some of the convenient
15//! functionality of the [`Rng`] trait, including [`gen`], [`gen_range`] and
16//! of course [`sample`].
17//!
18//! Abstractly, a [probability distribution] describes the probability of
19//! occurance of each value in its sample space.
20//!
21//! More concretely, an implementation of `Distribution<T>` for type `X` is an
22//! algorithm for choosing values from the sample space (a subset of `T`)
23//! according to the distribution `X` represents, using an external source of
24//! randomness (an RNG supplied to the `sample` function).
25//!
26//! A type `X` may implement `Distribution<T>` for multiple types `T`.
27//! Any type implementing [`Distribution`] is stateless (i.e. immutable),
28//! but it may have internal parameters set at construction time (for example,
29//! [`Uniform`] allows specification of its sample space as a range within `T`).
30//!
31//!
32//! # The `Standard` distribution
33//!
34//! The [`Standard`] distribution is important to mention. This is the
35//! distribution used by [`Rng::gen()`] and represents the "default" way to
36//! produce a random value for many different types, including most primitive
37//! types, tuples, arrays, and a few derived types. See the documentation of
38//! [`Standard`] for more details.
39//!
40//! Implementing `Distribution<T>` for [`Standard`] for user types `T` makes it
41//! possible to generate type `T` with [`Rng::gen()`], and by extension also
42//! with the [`random()`] function.
43//!
44//!
45//! # Distribution to sample from a `Uniform` range
46//!
47//! The [`Uniform`] distribution is more flexible than [`Standard`], but also
48//! more specialised: it supports fewer target types, but allows the sample
49//! space to be specified as an arbitrary range within its target type `T`.
50//! Both [`Standard`] and [`Uniform`] are in some sense uniform distributions.
51//!
52//! Values may be sampled from this distribution using [`Rng::gen_range`] or
53//! by creating a distribution object with [`Uniform::new`],
54//! [`Uniform::new_inclusive`] or `From<Range>`. When the range limits are not
55//! known at compile time it is typically faster to reuse an existing
56//! distribution object than to call [`Rng::gen_range`].
57//!
58//! User types `T` may also implement `Distribution<T>` for [`Uniform`],
59//! although this is less straightforward than for [`Standard`] (see the
60//! documentation in the [`uniform` module]. Doing so enables generation of
61//! values of type `T` with  [`Rng::gen_range`].
62//!
63//!
64//! # Other distributions
65//!
66//! There are surprisingly many ways to uniformly generate random floats. A
67//! range between 0 and 1 is standard, but the exact bounds (open vs closed)
68//! and accuracy differ. In addition to the [`Standard`] distribution Rand offers
69//! [`Open01`] and [`OpenClosed01`]. See [Floating point implementation] for
70//! more details.
71//!
72//! [`Alphanumeric`] is a simple distribution to sample random letters and
73//! numbers of the `char` type; in contrast [`Standard`] may sample any valid
74//! `char`.
75//!
76//!
77//! # Non-uniform probability distributions
78//!
79//! Rand currently provides the following probability distributions:
80//!
81//! - Related to real-valued quantities that grow linearly
82//!   (e.g. errors, offsets):
83//!   - [`Normal`] distribution, and [`StandardNormal`] as a primitive
84//!   - [`Cauchy`] distribution
85//! - Related to Bernoulli trials (yes/no events, with a given probability):
86//!   - [`Binomial`] distribution
87//!   - [`Bernoulli`] distribution, similar to [`Rng::gen_bool`].
88//! - Related to positive real-valued quantities that grow exponentially
89//!   (e.g. prices, incomes, populations):
90//!   - [`LogNormal`] distribution
91//! - Related to the occurrence of independent events at a given rate:
92//!   - [`Poisson`] distribution
93//!   - [`Exp`]onential distribution, and [`Exp1`] as a primitive
94//! - Gamma and derived distributions:
95//!   - [`Gamma`] distribution
96//!   - [`ChiSquared`] distribution
97//!   - [`StudentT`] distribution
98//!   - [`FisherF`] distribution
99//!
100//!
101//! # Examples
102//!
103//! Sampling from a distribution:
104//!
105//! ```
106//! use rand::{thread_rng, Rng};
107//! use rand::distributions::Exp;
108//!
109//! let exp = Exp::new(2.0);
110//! let v = thread_rng().sample(exp);
111//! println!("{} is from an Exp(2) distribution", v);
112//! ```
113//!
114//! Implementing the [`Standard`] distribution for a user type:
115//!
116//! ```
117//! # #![allow(dead_code)]
118//! use rand::Rng;
119//! use rand::distributions::{Distribution, Standard};
120//!
121//! struct MyF32 {
122//!     x: f32,
123//! }
124//!
125//! impl Distribution<MyF32> for Standard {
126//!     fn sample<R: Rng + ?Sized>(&self, rng: &mut R) -> MyF32 {
127//!         MyF32 { x: rng.gen() }
128//!     }
129//! }
130//! ```
131//!

302/// Types (distributions) that can be used to create a random instance of `T`.
303///
304/// It is possible to sample from a distribution through both the
305/// `Distribution` and [`Rng`] traits, via `distr.sample(&mut rng)` and
306/// `rng.sample(distr)`. They also both offer the [`sample_iter`] method, which
307/// produces an iterator that samples from the distribution.
308///
309/// All implementations are expected to be immutable; this has the significant
310/// advantage of not needing to consider thread safety, and for most
311/// distributions efficient state-less sampling algorithms are available.
312///
313/// [`Rng`]: ../trait.Rng.html
314/// [`sample_iter`]: trait.Distribution.html#method.sample_iter
315pub trait Distribution<T> {
316    /// Generate a random value of `T`, using `rng` as the source of randomness.
317    fn sample<R: Rng + ?Sized>(&self, rng: &mut R) -> T;
318
319    /// Create an iterator that generates random values of `T`, using `rng` as
320    /// the source of randomness.
321    ///
322    /// # Example
323    ///
324    /// ```
325    /// use rand::thread_rng;
326    /// use rand::distributions::{Distribution, Alphanumeric, Uniform, Standard};
327    ///
328    /// let mut rng = thread_rng();
329    ///
330    /// // Vec of 16 x f32:
331    /// let v: Vec<f32> = Standard.sample_iter(&mut rng).take(16).collect();
332    ///
333    /// // String:
334    /// let s: String = Alphanumeric.sample_iter(&mut rng).take(7).collect();
335    ///
336    /// // Dice-rolling:
337    /// let die_range = Uniform::new_inclusive(1, 6);
338    /// let mut roll_die = die_range.sample_iter(&mut rng);
339    /// while roll_die.next().unwrap() != 6 {
340    ///     println!("Not a 6; rolling again!");
341    /// }
342    /// ```
343    fn sample_iter<'a, R>(&'a self, rng: &'a mut R) -> DistIter<'a, Self, R, T>
344        where Self: Sized, R: Rng
345    {
346        DistIter {
347            distr: self,
348            rng: rng,
349            phantom: ::core::marker::PhantomData,
350        }
351    }
352}
353
354impl<'a, T, D: Distribution<T>> Distribution<T> for &'a D {
355    fn sample<R: Rng + ?Sized>(&self, rng: &mut R) -> T {
356        (*self).sample(rng)
357    }
358}

361/// An iterator that generates random values of `T` with distribution `D`,
362/// using `R` as the source of randomness.
363///
364/// This `struct` is created by the [`sample_iter`] method on [`Distribution`].
365/// See its documentation for more.
366///
367/// [`Distribution`]: trait.Distribution.html
368/// [`sample_iter`]: trait.Distribution.html#method.sample_iter
369#[derive(Debug)]
370pub struct DistIter<'a, D: 'a, R: 'a, T> {
371    distr: &'a D,
372    rng: &'a mut R,
373    phantom: ::core::marker::PhantomData<T>,
374}

376impl<'a, D, R, T> Iterator for DistIter<'a, D, R, T>
377    where D: Distribution<T>, R: Rng + 'a
378{
379    type Item = T;
380

383        Some(self.distr.sample(self.rng))

387        (usize::max_value(), None)
388    }

392/// A generic random value distribution, implemented for many primitive types.
393/// Usually generates values with a numerically uniform distribution, and with a
394/// range appropriate to the type.
395/// 
396/// ## Built-in Implementations
397///
398/// Assuming the provided `Rng` is well-behaved, these implementations
399/// generate values with the following ranges and distributions:
400///
401/// * Integers (`i32`, `u32`, `isize`, `usize`, etc.): Uniformly distributed
402///   over all values of the type.
403/// * `char`: Uniformly distributed over all Unicode scalar values, i.e. all
404///   code points in the range `0...0x10_FFFF`, except for the range
405///   `0xD800...0xDFFF` (the surrogate code points). This includes
406///   unassigned/reserved code points.
407/// * `bool`: Generates `false` or `true`, each with probability 0.5.
408/// * Floating point types (`f32` and `f64`): Uniformly distributed in the
409///   half-open range `[0, 1)`. See notes below.
410/// * Wrapping integers (`Wrapping<T>`), besides the type identical to their
411///   normal integer variants.
412///
413/// The following aggregate types also implement the distribution `Standard` as
414/// long as their component types implement it:
415///
416/// * Tuples and arrays: Each element of the tuple or array is generated
417///   independently, using the `Standard` distribution recursively.
418/// * `Option<T>` where `Standard` is implemented for `T`: Returns `None` with
419///   probability 0.5; otherwise generates a random `x: T` and returns `Some(x)`.
420///
421/// # Example
422/// ```
423/// use rand::prelude::*;
424/// use rand::distributions::Standard;
425///
426/// let val: f32 = SmallRng::from_entropy().sample(Standard);
427/// println!("f32 from [0, 1): {}", val);
428/// ```
429///
430/// # Floating point implementation
431/// The floating point implementations for `Standard` generate a random value in
432/// the half-open interval `[0, 1)`, i.e. including 0 but not 1.
433///
434/// All values that can be generated are of the form `n * ε/2`. For `f32`
435/// the 23 most significant random bits of a `u32` are used and for `f64` the
436/// 53 most significant bits of a `u64` are used. The conversion uses the
437/// multiplicative method: `(rng.gen::<$uty>() >> N) as $ty * (ε/2)`.
438///
439/// See also: [`Open01`] which samples from `(0, 1)`, [`OpenClosed01`] which
440/// samples from `(0, 1]` and `Rng::gen_range(0, 1)` which also samples from
441/// `[0, 1)`. Note that `Open01` and `gen_range` (which uses [`Uniform`]) use
442/// transmute-based methods which yield 1 bit less precision but may perform
443/// faster on some architectures (on modern Intel CPUs all methods have
444/// approximately equal performance).
445///
446/// [`Open01`]: struct.Open01.html
447/// [`OpenClosed01`]: struct.OpenClosed01.html
448/// [`Uniform`]: uniform/struct.Uniform.html
449#[derive(Clone, Copy, Debug)]
450pub struct Standard;
451
452#[allow(deprecated)]
453impl<T> ::Rand for T where Standard: Distribution<T> {
454    fn rand<R: Rng>(rng: &mut R) -> Self {
455        Standard.sample(rng)
456    }
457}

460/// A value with a particular weight for use with `WeightedChoice`.
461#[derive(Copy, Clone, Debug)]
462pub struct Weighted<T> {
463    /// The numerical weight of this item
464    pub weight: u32,
465    /// The actual item which is being weighted
466    pub item: T,
467}
468
469/// A distribution that selects from a finite collection of weighted items.
470///
471/// Each item has an associated weight that influences how likely it
472/// is to be chosen: higher weight is more likely.
473///
474/// The `Clone` restriction is a limitation of the `Distribution` trait.
475/// Note that `&T` is (cheaply) `Clone` for all `T`, as is `u32`, so one can
476/// store references or indices into another vector.
477///
478/// # Example
479///
480/// ```
481/// use rand::distributions::{Weighted, WeightedChoice, Distribution};
482///
483/// let mut items = vec!(Weighted { weight: 2, item: 'a' },
484///                      Weighted { weight: 4, item: 'b' },
485///                      Weighted { weight: 1, item: 'c' });
486/// let wc = WeightedChoice::new(&mut items);
487/// let mut rng = rand::thread_rng();
488/// for _ in 0..16 {
489///      // on average prints 'a' 4 times, 'b' 8 and 'c' twice.
490///      println!("{}", wc.sample(&mut rng));
491/// }
492/// ```
493#[derive(Debug)]
494pub struct WeightedChoice<'a, T:'a> {
495    items: &'a mut [Weighted<T>],
496    weight_range: Uniform<u32>,
497}
498
499impl<'a, T: Clone> WeightedChoice<'a, T> {
500    /// Create a new `WeightedChoice`.
501    ///
502    /// Panics if:
503    ///
504    /// - `items` is empty
505    /// - the total weight is 0
506    /// - the total weight is larger than a `u32` can contain.
507    pub fn new(items: &'a mut [Weighted<T>]) -> WeightedChoice<'a, T> {
508        // strictly speaking, this is subsumed by the total weight == 0 case
509        assert!(!items.is_empty(), "WeightedChoice::new called with no items");
510
511        let mut running_total: u32 = 0;
512
513        // we convert the list from individual weights to cumulative
514        // weights so we can binary search. This *could* drop elements
515        // with weight == 0 as an optimisation.
516        for item in items.iter_mut() {
517            running_total = match running_total.checked_add(item.weight) {
518                Some(n) => n,
519                None => panic!("WeightedChoice::new called with a total weight \
520                               larger than a u32 can contain")
521            };
522
523            item.weight = running_total;
524        }
525        assert!(running_total != 0, "WeightedChoice::new called with a total weight of 0");
526
527        WeightedChoice {
528            items,
529            // we're likely to be generating numbers in this range
530            // relatively often, so might as well cache it
531            weight_range: Uniform::new(0, running_total)
532        }
533    }
534}
535
536impl<'a, T: Clone> Distribution<T> for WeightedChoice<'a, T> {
537    fn sample<R: Rng + ?Sized>(&self, rng: &mut R) -> T {
538        // we want to find the first element that has cumulative
539        // weight > sample_weight, which we do by binary since the
540        // cumulative weights of self.items are sorted.
541
542        // choose a weight in [0, total_weight)
543        let sample_weight = self.weight_range.sample(rng);
544
545        // short circuit when it's the first item
546        if sample_weight < self.items[0].weight {
547            return self.items[0].item.clone();
548        }
549
550        let mut idx = 0;
551        let mut modifier = self.items.len();
552
553        // now we know that every possibility has an element to the
554        // left, so we can just search for the last element that has
555        // cumulative weight <= sample_weight, then the next one will
556        // be "it". (Note that this greatest element will never be the
557        // last element of the vector, since sample_weight is chosen
558        // in [0, total_weight) and the cumulative weight of the last
559        // one is exactly the total weight.)
560        while modifier > 1 {
561            let i = idx + modifier / 2;
562            if self.items[i].weight <= sample_weight {
563                // we're small, so look to the right, but allow this
564                // exact element still.
565                idx = i;
566                // we need the `/ 2` to round up otherwise we'll drop
567                // the trailing elements when `modifier` is odd.
568                modifier += 1;
569            } else {
570                // otherwise we're too big, so go left. (i.e. do
571                // nothing)
572            }
573            modifier /= 2;
574        }
575        self.items[idx + 1].item.clone()
576    }
577}

579/// Sample a random number using the Ziggurat method (specifically the
580/// ZIGNOR variant from Doornik 2005). Most of the arguments are
581/// directly from the paper:
582///
583/// * `rng`: source of randomness
584/// * `symmetric`: whether this is a symmetric distribution, or one-sided with P(x < 0) = 0.
585/// * `X`: the $x_i$ abscissae.
586/// * `F`: precomputed values of the PDF at the $x_i$, (i.e. $f(x_i)$)
587/// * `F_DIFF`: precomputed values of $f(x_i) - f(x_{i+1})$
588/// * `pdf`: the probability density function
589/// * `zero_case`: manual sampling from the tail when we chose the
590///    bottom box (i.e. i == 0)
591
592// the perf improvement (25-50%) is definitely worth the extra code
593// size from force-inlining.
594#[cfg(feature="std")]
595#[inline(always)]
596fn ziggurat<R: Rng + ?Sized, P, Z>(
597            rng: &mut R,
598            symmetric: bool,
599            x_tab: ziggurat_tables::ZigTable,
600            f_tab: ziggurat_tables::ZigTable,
601            mut pdf: P,
602            mut zero_case: Z)
603            -> f64 where P: FnMut(f64) -> f64, Z: FnMut(&mut R, f64) -> f64 {
604    loop {
605        // As an optimisation we re-implement the conversion to a f64.
606        // From the remaining 12 most significant bits we use 8 to construct `i`.
607        // This saves us generating a whole extra random number, while the added
608        // precision of using 64 bits for f64 does not buy us much.
609        let bits = rng.next_u64();
610        let i = bits as usize & 0xff;
611
612        let u = if symmetric {
613            // Convert to a value in the range [2,4) and substract to get [-1,1)
614            // We can't convert to an open range directly, that would require
615            // substracting `3.0 - EPSILON`, which is not representable.
616            // It is possible with an extra step, but an open range does not
617            // seem neccesary for the ziggurat algorithm anyway.
618            (bits >> 12).into_float_with_exponent(1) - 3.0
619        } else {
620            // Convert to a value in the range [1,2) and substract to get (0,1)
621            (bits >> 12).into_float_with_exponent(0)
622            - (1.0 - ::core::f64::EPSILON / 2.0)
623        };
624        let x = u * x_tab[i];
625
626        let test_x = if symmetric { x.abs() } else {x};
627
628        // algebraically equivalent to |u| < x_tab[i+1]/x_tab[i] (or u < x_tab[i+1]/x_tab[i])
629        if test_x < x_tab[i + 1] {
630            return x;
631        }
632        if i == 0 {
633            return zero_case(rng, u);
634        }
635        // algebraically equivalent to f1 + DRanU()*(f0 - f1) < 1
636        if f_tab[i + 1] + (f_tab[i] - f_tab[i + 1]) * rng.gen::<f64>() < pdf(x) {
637            return x;
638        }
639    }
640}

656                let mut items = $items;
657                let mut total_weight = 0;
658                for item in &items { total_weight += item.weight; }
659
660                let wc = WeightedChoice::new(&mut items);
661                let expected = $expected;
662
663                // Use extremely large steps between the random numbers, because
664                // we test with small ranges and `UniformInt` is designed to prefer
665                // the most significant bits.
666                let mut rng = StepRng::new(0, !0 / (total_weight as u64));

668                for &val in expected.iter() {

671            }}
672        }
